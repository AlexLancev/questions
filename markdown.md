# 1. База JavaScript

### 1. Что такое JavaScript?

Это высокоуровневый, интерпретируемый (с JIT-компиляцией), однопоточный, мультипарадигмальный (ООП на прототипах + функциональный) язык программирования.

`ECMAScript (ES)` — это спека, которая описывает, как должен работать язык сценариев. Она определяет синтаксис, типы данных, инструкции и встроенные объекты. Самыми фундаментальными обновами считаются ES5 (2009) и ES2015 (ES6). С 2016 года стандарты стали выпускатся регулярно.

`Движок JS` (V8, SpiderMonkey, JavaScriptCore и др.) умеет читать код и выполнять инструкции в точности так, как предписано спекой ES. При этом движок — это только ядро: он выполняет код, управляет компиляцией, стеком вызовов и кучей памяти, но ничего не знает ни о таймерах, ни о сети, ни о событиях.

`Хост-среда` (браузер, Node.js, Bun) предоставляет всё необходимое для работы с внешним миром: Web API, Event Loop, очереди задач, модули (fs, http), DOM, fetch и т.д.

### 2. var vs let vs const
- `var` — функциональная область видимости, всплывает со значением `undefined`, позволяет переобъявление. 
- `let`/`const` — блочная видимость, всплывает со значением `uninitialized`, но находятся в Temporal Dead Zone до объявления, `const` запрещает переопределение. В проде используем только `const` (по умолчанию) и `let`.

### 3. Scope (Область видимости)
Контекст, в котором доступны переменные. Бывает глобальным, функциональным и блочным. JS использует лексическую область видимости (определяется в момент написания кода, а не вызова).

### 4. Hoisting (Всплытие)
Механизм, при котором объявления переменных и функций поднимаются наверх своей области видимости на этапе компиляции. Функции (declaration) всплывают целиком. 

`var`/`params` — всплывает со значением `undefined`.
`let`/`const`/`class` —  всплывает со значением `uninitialized`,

### 5. Strict mode
Режим `"use strict"`, который исправляет ошибки дизайна языка (запрещает создание глобальных переменных по ошибке, дублирование параметров функций и т.д.). В ES-модулях включен по умолчанию.

### 6. Примитивные типы
`string`, `number`, `boolean`, `null`, `undefined`, `symbol`, `bigint`.

### 7. null vs undefined
`undefined` — значение не присвоено (дефолт языка). `null` — намеренное отсутствие значения (присваивается разработчиком).

### 8. typeof
Оператор для определения типа. Важный нюанс: `typeof null === "object"` (ошибка языка) и `typeof function(){} === "function"`.

### 9. == vs ===
`==` сравнивает с приведением типов (абстрактное равенство), `===` без приведения (строгое). В проде всегда `===`, чтобы избежать побочных эффектов.

### 10. Truthy и Falsy
Falsy значения: `false`, `0`, `""`, `null`, `undefined`, `NaN`. Все остальное — Truthy (включая пустые массивы `[]` и объекты `{}`).

# 2. Функции

### 11. Разница Declaration vs Expression
Declaration (`function a(){}`) всплывает, ее можно вызвать до объявления. Expression (`const a = function(){}`) не всплывает (зависит от переменной).

### 12. Расскижите Arrow function
Короткий синтаксис. Главные отличия: не имеет своего `this`, `arguments`, `super` и не может быть конструктором (`new`). `this` берется из внешнего лексического окружения.

### 13. Расскижите про arguments у функции
Итерируемый объект (не массив), доступный внутри обычных функций, содержащий все переданные аргументы. В современном коде заменяется на `...rest`.

### 14. Зачем нужен оператор return
Завершает выполнение функции и возвращает значение. Если не указать — вернется `undefined`.

### 15. Расскижите про Callback
Функция, переданная в другую функцию как аргумент для вызова позже (основа асинхронности в JS).

### 16. Расскижите про Pure function (Чистая функция)
1. Детерминирована (один и тот же вход — всегда один и тот же выход).
2. Без побочных эффектов (не меняет глобальные переменные, API, логи).

### 17. Расскижите про IIFE
Immediately Invoked Function Expression. Функция, которая выполняется сразу после объявления. Раньше использовалась для инкапсуляции (создания приватности), сейчас почти вытеснена модулями.

### 18. Расскижите про Rest parameters (...args)
Позволяет собрать неопределенное количество аргументов в настоящий массив.

### 19. Расскижите про Default parameters
Позволяют задать значения аргументам, если они не переданы или равны `undefined`.

### 20. Функция как аргумент
Да, в JS функции — это «объекты первого класса» (first-class citizens). Их можно хранить в переменных, передавать и возвращать.

# 3. Объекты

### 21. Что такое объект?
Ссылочный тип данных, представляющий собой коллекцию пар «ключ-значение».

### 22. Как создать объект?
Литерал `{}`, конструктор `new Object()`, `Object.create()`, или через `class`.

### 23. Разница Object.create(null) vs {}
`{}` наследует методы `Object.prototype` (например, `toString`). `Object.create(null)` создает абсолютно пустой объект без прототипа.

### 24. Образно про методы объекта
Это функции, закрепленные за свойствами объекта.

### 25. На что ссылается this?
Ссылается на объект, в контексте которого выполняется текущий код.

## 26. От чего зависит значение this?
В JavaScript `this` определяется в **момент вызова функции** (динамическое связывание), а не в момент объявления. Исключение — стрелочные функции (у них `this` лексический). Значение зависит от:
- Вызова от имени объекта: `obj.method()`
- Явного связывания: `call`, `apply`, `bind`
- Конструктора: `new`
- Глобального контекста (в strict mode — `undefined`, без него — `window`/`global`)

## 27. Что делают методы call, apply, bind?
Используются для явной установки `this`.
- **call**(thisArg, arg1, arg2) — вызывает функцию немедленно, аргументы через запятую
- **apply**(thisArg, [args]) — вызывает немедленно, аргументы массивом
- **bind**(thisArg) — возвращает новую функцию с привязанным контекстом, которую можно вызвать позже

## 28. Как скопировать объект?
- **Поверхностно**: `Object.assign({}, obj)` или spread-оператор `{...obj}`
- **Глубоко**: `structuredClone(obj)` (современный стандарт), `JSON.parse(JSON.stringify(obj))` (с ограничениями) или библиотеки вроде lodash.cloneDeep

## 29. Что делает Object.assign()?
Копирует все перечисляемые собственные свойства из одного или нескольких исходных объектов в целевой. Возвращает измененный целевой объект.

## 30. Чем поверхностное копирование отличается от глубокого?
**Поверхностное** копирует только значения первого уровня. Если значением является ссылка на другой объект/массив, скопируется именно ссылка (оба объекта будут смотреть на одни и те же данные в памяти).

**Глубокое** копирование рекурсивно создает дубликаты всех вложенных структур.

# 4. Массивы

## 31. Чем массив отличается от объекта?
Массив в JS — это особый тип объекта, предназначенный для упорядоченных данных. У него есть свойство `length`, числовые индексы в качестве ключей и встроенные методы для итерации (`map`, `push` и т.д.).

## 32. Методы map, filter, reduce:
- **map**: создает новый массив, применяя функцию к каждому элементу
- **filter**: создает новый массив из элементов, прошедших проверку
- **reduce**: сворачивает массив в одно значение (число, строку, объект, другой массив), используя аккумулятор

## 33. Чем forEach отличается от map?
**forEach** просто выполняет функцию для каждого элемента (используется для побочных эффектов) и возвращает `undefined`.

**map** всегда возвращает новый массив той же длины.

## 34. Что делает find?
Возвращает первый элемент массива, который удовлетворяет условию. Если ничего не найдено — `undefined`.

## 35. Что делает some и every?
- **some**: возвращает булевое значение, `true` - если хотя бы один элемент прошел проверку
- **every**: возвращает булевое значение, `true` - если все элементы прошли проверку

Оба метода прекращают выполнение, как только результат становится ясен (short-circuiting).

## 36. Как удалить элемент из массива?
- `shift()` (с начала) (мутирует исходный массив),
- `pop()` (с конца) (мутирует исходный массив), 
- `splice(index, count)` (мутирует исходный массив),
- `filter` (не мутируя исходный массив).

## 37. Что делает splice?
Универсальный метод: удаляет, заменяет или добавляет элементы внутри исходного массива (мутирует его).

## 38. Что делает slice?
Возвращает копию части массива (с индекса по индекс, не включая последний). Не мутирует оригинал.

## 39. Как объединить массивы?
`concat()` или spread-оператор: `[...arr1, ...arr2]`.

## 40. Как убрать дубликаты из массива?
Самый быстрый способ: `[...new Set(array)]`. Для объектов в массиве — через `filter` с проверкой индекса или `Map`.

## 40.1 Назовите мутирующие / не мутирующие методы массивов?
- **Мутируют -** push, pop, shift, unshift, splice, sort, reverse, fill

- **Не мутируют -** map, filter, concat, slice, reduce

# 5. Асинхронность (Junior → Middle)

## 41. Что такое event loop?
Механизм, который позволяет JavaScript выполнять асинхронные операции, несмотря на однопоточность. Он постоянно проверяет Call Stack и, если тот пуст, переносит задачи из очередей (микро- и макротаски) в стек.

## 42. Что такое call stack?
Структура данных (LIFO — Last In, First Out), которая хранит информацию о текущих вызовах функций. Когда функция вызывается, она попадает в стек, когда завершается — удаляется.

## 43. Что такое callback hell?
Ситуация с глубокой вложенностью колбэков, которая делает код нечитаемым и сложным в отладке. Решается промисами или `async/await`.

## 44. Что такое Promise?
Объект, представляющий результат асинхронной операции, которая еще не завершена. Позволяет писать асинхронный код в линейном стиле.

## 45. Какие состояния у Promise?
1. **pending** (ожидание)
2. **fulfilled** (выполнено успешно)
3. **rejected** (выполнено с ошибкой)

## 46. Что делает .then()?
Принимает два колбэка: для успеха и (опционально) для ошибки. Всегда возвращает новый промис, что позволяет строить цепочки (chaining).

## 47. Что делает .catch()?
Служит для обработки ошибок в цепочке промисов. По сути, это сахар над `.then(null, rejectionHandler)`.

## 48. Что делает .finally()?
Выполняется в любом случае (успех или ошибка). Удобно для очистки ресурсов (например, выключения лоадера в UI).

## 49. Что такое async/await?
Синтаксический сахар над промисами, позволяющий писать асинхронный код так, будто он синхронный. `async` функция всегда возвращает промис.

## 50. Чем async/await отличается от Promise?
- **Читаемость**: код выглядит чище, нет вложенности `.then()`
- **Обработка ошибок**: можно использовать стандартный `try/catch`
- **Отладка**: стек вызовов в `async/await` обычно информативнее
